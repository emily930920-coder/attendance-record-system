## 快速入门

### 第一次使用

1. **启动服务器**
   - Mac/Linux: 运行 `./start.sh`
   - Windows: 双击 `start.bat`
   - 或者直接运行: `npm start`

2. **访问系统**
   - 打开浏览器访问: http://localhost:3000
   - 看到界面后，系统就可以使用了！

3. **设置用户名**
   - 点击页面顶部的"切换用户"按钮
   - 输入你的姓名或工号
   - 点击确定

4. **开始打卡**
   - 点击"上班打卡"按钮记录上班时间
   - 工作结束后点击"下班打卡"按钮
   - 系统会自动计算工作时长

## 功能详解

### 1. 打卡功能

#### 上班打卡
- **何时使用**: 开始工作时
- **操作**: 点击"上班打卡"按钮
- **效果**: 记录当前时间为上班时间
- **注意**: 每天只能打一次上班卡

#### 下班打卡
- **何时使用**: 结束工作时
- **操作**: 点击"下班打卡"按钮
- **效果**: 记录当前时间为下班时间，自动计算工作时长
- **注意**: 必须先打上班卡才能打下班卡

### 2. 记录查看

#### 今日记录
显示今天的打卡情况：
- 上班时间
- 下班时间
- 工作时长（自动计算）

#### 本月统计
显示本月的汇总数据：
- **工作天数**: 本月打过卡的天数
- **总工时**: 本月累计工作小时数
- **平均工时**: 平均每天工作小时数

### 3. 历史记录管理

#### 查看历史
- 在"本月记录"区域查看本月所有记录
- 记录按日期降序排列（最新的在上面）
- 显示每天的上班时间、下班时间和工作时长

#### 手动添加记录
如果忘记打卡，可以手动补录：

1. 点击"手动添加"按钮
2. 选择日期
3. 输入上班时间
4. 输入下班时间（可选）
5. 点击确定

#### 编辑记录
如果打卡时间不准确：

1. 找到要修改的记录
2. 点击"编辑"按钮
3. 修改时间
4. 点击保存

#### 删除记录
如果需要删除错误的记录：

1. 找到要删除的记录
2. 点击"删除"按钮
3. 确认删除

### 4. 多用户功能

#### 适用场景
- 家庭成员共用一台电脑
- 小团队使用
- 管理多个工作的时间记录

#### 使用方法
1. 点击页面顶部的"切换用户"按钮
2. 输入新的用户名
3. 点击确定
4. 系统会切换到该用户的数据

#### 数据隔离
- 每个用户的数据完全独立
- 切换用户不会丢失数据
- 可以随时切换回之前的用户

### 5. 离线模式

#### 自动离线支持
系统支持 PWA（渐进式 Web 应用）：
- 首次访问后自动缓存
- 离线状态下可以正常使用
- 离线数据保存在本地
- 恢复网络后自动同步

#### 手动同步
如果数据未自动同步：
1. 点击"同步数据"按钮
2. 等待同步完成
3. 查看同步状态提示

### 6. 数据导出

#### 导出个人数据
1. 点击"导出数据"按钮
2. 选择保存位置
3. 文件格式: JSON
4. 文件名: `工时记录_用户名_日期.json`

#### 导出数据用途
- 备份数据
- 提交给公司人事
- 数据分析
- 迁移到其他系统

## 常见问题

### Q1: 忘记打卡怎么办？
**A:** 使用"手动添加"功能补录记录。

### Q2: 打卡时间错了怎么办？
**A:** 点击记录右侧的"编辑"按钮修改时间。

### Q3: 数据会丢失吗？
**A:** 不会。数据同时保存在：
- 服务器文件系统中
- 浏览器本地存储中
建议定期导出数据备份。

### Q4: 多人使用会不会数据混乱？
**A:** 不会。每个用户的数据完全独立，互不影响。

### Q5: 离线状态下打卡有效吗？
**A:** 有效。离线打卡会保存在本地，恢复网络后自动同步到服务器。

### Q6: 如何在手机上使用？
**A:** 
1. 确保手机和电脑在同一局域网
2. 在浏览器中访问电脑的IP地址:3000
3. 建议添加到主屏幕作为应用使用

### Q7: 可以修改过去的记录吗？
**A:** 可以。找到对应记录，点击"编辑"按钮即可修改。

### Q8: 工作时长是如何计算的？
**A:** 下班时间 - 上班时间 = 工作时长（以小时为单位）

### Q9: 支持跨月查询吗？
**A:** 当前版本只显示本月记录。可以通过导出数据查看历史月份。

### Q10: 服务器关闭后数据会丢失吗？
**A:** 不会。数据保存在 `server/data` 目录的JSON文件中，服务器重启后数据仍然存在。

## 最佳实践

### 日常使用建议

1. **养成打卡习惯**
   - 到达办公室/开始工作: 立即打上班卡
   - 离开办公室/结束工作: 立即打下班卡
   - 在浏览器中收藏该页面，方便访问

2. **定期检查记录**
   - 每周检查一次打卡记录
   - 及时补录遗漏的记录
   - 修正错误的时间

3. **数据备份**
   - 每月月底导出数据
   - 保存到云盘或本地硬盘
   - 建议保留至少半年的数据

4. **多设备使用**
   - 将服务器地址添加到手机浏览器收藏
   - 在电脑和手机上都可以打卡
   - 数据实时同步，任意设备查看

### 团队使用建议

1. **服务器部署**
   - 使用局域网服务器部署
   - 或使用云服务器提供外网访问
   - 确保服务器稳定运行

2. **用户规范**
   - 统一用户名格式（如：姓名、工号）
   - 制定打卡规则
   - 定期导出汇总数据

3. **数据管理**
   - 定期备份 `server/data` 目录
   - 监控服务器运行状态
   - 必要时设置访问密码

### 进阶使用技巧

1. **快速访问**
   - 手机: 添加到主屏幕
   - 电脑: 浏览器收藏夹或快捷方式
   - 设置为浏览器主页

2. **数据分析**
   - 导出JSON数据
   - 使用Excel或Python分析
   - 生成个性化报表

3. **自定义部署**
   - 修改端口号（编辑 server/server.js）
   - 自定义样式（编辑 public/style.css）
   - 添加更多功能（编辑 public/app.js）
