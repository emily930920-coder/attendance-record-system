### 可能的问题

#### 1. 没有触发备份操作
**原因**: 还没有进行打卡操作

**解决**: 访问 http://localhost:3000 并进行打卡

#### 2. 服务器需要重启
**原因**: dotenv配置后需要重启才能生效

**解决**:
```bash
# 停止服务器 (Ctrl+C 或)
lsof -ti:3000 | xargs kill -9

# 重启
cd /Users/yangyang100/Desktop/AutoTest/attendance-record-system
npm start
```

#### 3. Discord频道问题
**原因**: Webhook消息可能被过滤或隐藏

**解决**: 
- 检查Discord频道设置
- 确认没有消息过滤规则
- 尝试发送测试消息

## 确认备份是否触发

### 方法1: 查看服务器日志
服务器日志应该显示：
```
🔄 实时备份 - 用户: yangyang100
✅ Discord备份成功
```

或：
```
📦 处理备份队列: 1 个用户
🔄 实时备份 - 用户: yangyang100
```

### 方法2: 检查备份状态API
```bash
curl http://localhost:3000/api/backup/status
```

应该返回：
```json
{
  "enabled": true,
  "hasWebhook": false,
  "hasDiscord": true,
  "queueLength": 0,
  "message": "实时备份已启用"
}
```

### 方法3: 查看Discord频道
直接检查Discord频道是否有新消息

## 完整测试流程

1. **确认服务器运行**
   ```bash
   lsof -i:3000
   # 应该看到node进程
   ```

2. **确认配置加载**
   ```bash
   curl http://localhost:3000/api/backup/status
   # hasDiscord应该为true
   ```

3. **发送测试消息**
   ```bash
   node test-discord.js
   # 检查Discord是否收到
   ```

4. **真实打卡测试**
   - 打开 http://localhost:3000
   - 点击打卡按钮
   - 等待5秒
   - 检查Discord

5. **查看日志**
   ```bash
   # 查看最新日志
   tail -f server日志文件
   ```

## 如果仍然失败

### 检查node-fetch版本
```bash
npm list node-fetch
# 应该是2.x版本
```

### 重新安装依赖
```bash
rm -rf node_modules package-lock.json
npm install
npm start
```

### 使用Webhook.site备用
如果Discord一直不行，可以改用Webhook.site：

1. 访问 https://webhook.site
2. 复制URL
3. 修改 .env:
   ```
   BACKUP_WEBHOOK_URL=https://webhook.site/你的ID
   ```
4. 重启服务器
