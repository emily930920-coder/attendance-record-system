# 📋 备份服务配置指南

## 方式1: Webhook.site（最简单，推荐）

### 1. 获取Webhook URL

1. 访问: https://webhook.site
2. 页面会自动生成一个唯一的URL，类似：
   ```
   https://webhook.site/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
   ```
3. **复制这个URL**（保持页面打开，可以实时查看备份）

### 2. 配置到本地环境

```bash
# 打开终端，执行：
export BACKUP_WEBHOOK_URL="https://webhook.site/你的唯一ID"

# 验证配置
echo $BACKUP_WEBHOOK_URL
```

### 3. 重启服务器

```bash
cd /Users/yangyang100/Desktop/AutoTest/attendance-record-system
npm start
```

启动后会显示：
```
🔄 实时备份配置:
  ✅ Webhook备份已配置
  💾 每次数据变更将自动备份
```

### 4. 测试备份

1. 访问: http://localhost:3000
2. 进行打卡操作
3. 返回webhook.site页面
4. 应该能看到备份请求，包含完整的JSON数据

### 5. 查看备份数据

在webhook.site页面：
- **Request body**: 完整的备份数据
- **Download**: 可下载JSON文件
- **History**: 查看所有历史备份

## 方式2: Discord Webhook（推荐，移动端可查看）

### 1. 创建Discord服务器和频道

如果没有Discord：
1. 下载Discord App: https://discord.com
2. 注册账号
3. 创建服务器（点击左侧"+"）
4. 创建频道，命名为"工时备份"

### 2. 获取Webhook URL

1. 右键点击频道
2. 编辑频道 → 整合
3. Webhooks → 新建Webhook
4. 自定义名称：工时记录系统
5. **复制Webhook URL**

### 3. 配置到本地环境

```bash
export DISCORD_WEBHOOK_URL="https://discord.com/api/webhooks/你的webhook-url"

# 或同时配置两个
export BACKUP_WEBHOOK_URL="https://webhook.site/xxx"
export DISCORD_WEBHOOK_URL="https://discord.com/api/webhooks/xxx"
```

### 4. 重启并测试

启动服务器，打卡测试，Discord频道会收到备份消息。

## 部署到Render时的配置

### 1. 在Render Dashboard配置

1. 登录 https://render.com
2. 选择你的服务
3. **Environment** 标签
4. 点击 **Add Environment Variable**

添加以下配置：

```
Key: BACKUP_WEBHOOK_URL
Value: https://webhook.site/你的唯一ID
```

或Discord：

```
Key: DISCORD_WEBHOOK_URL
Value: https://discord.com/api/webhooks/你的webhook-url
```

5. 点击 **Save Changes**
6. 服务会自动重新部署

### 2. 验证配置

部署完成后：
- 访问你的Render服务
- 进行打卡测试
- 查看webhook.site或Discord是否收到备份

### 一键配置脚本

创建 `.env` 文件：

```bash
cat > /Users/yangyang100/Desktop/AutoTest/attendance-record-system/.env << 'EOF'

```

然后修改 `server/server.js` 加载 `.env`：

```bash
npm install dotenv
```

## 常见问题

### Q: Webhook.site会过期吗？
A: 免费版30天自动删除数据，但URL永久有效。

### Q: 可以同时配置多个吗？
A: 可以！同时配置Webhook和Discord，双重保障。

### Q: 配置错了怎么办？
A: 直接修改环境变量，重启服务即可。

### Q: 如何停用备份？
A: 删除环境变量，或设为空字符串。
