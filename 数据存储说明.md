## 🔍 如何查看数据

### 方式1: 直接查看文件
```bash
# 查看所有数据文件
ls -lh server/data/*.json

# 查看具体用户的数据
cat server/data/yangyang100.json

# 美化JSON格式查看
cat server/data/yangyang100.json | python -m json.tool
```

### 方式2: 在系统中查看
1. 打开浏览器访问 http://localhost:3000
2. 切换到对应用户
3. 查看"本月记录"区域

### 方式3: 通过API查看
```bash
# 获取用户数据
curl http://localhost:3000/api/records/yangyang100

# 导出数据
curl http://localhost:3000/api/export/yangyang100 > backup.json
```

## 💾 数据管理

### 备份数据
```bash
# 备份整个数据目录
cp -r server/data ~/Desktop/attendance-backup-$(date +%Y%m%d)

# 备份单个用户
cp server/data/yangyang100.json ~/Desktop/yangyang100-backup.json
```

### 恢复数据
```bash
# 恢复整个数据目录
cp -r ~/Desktop/attendance-backup-20260430/* server/data/

# 恢复单个用户
cp ~/Desktop/yangyang100-backup.json server/data/yangyang100.json
```

### 清空数据
```bash
# 清空所有数据（保留.gitkeep）
rm server/data/*.json

# 删除特定用户数据
rm server/data/yangyang100.json
```

## 🔒 数据安全

### 数据存储特点
- ✅ 本地文件存储，数据完全在你的控制下
- ✅ 纯文本JSON格式，易于备份和迁移
- ✅ 每个用户独立文件，数据隔离
- ✅ 无需数据库，简单可靠

### 建议
1. **定期备份**: 每周备份一次数据目录
2. **版本控制**: 重要数据可提交到Git（注意.gitignore已排除）
3. **云端备份**: 可以将数据目录同步到云盘
4. **导出存档**: 每月底导出一次数据

## 📂 目录结构

```
server/data/
├── .gitkeep              # 确保目录被Git跟踪
├── default.json          # 默认用户数据
├── yangyang100.json      # yangyang100用户数据
└── [用户名].json         # 其他用户数据（自动创建）
```

## 🔄 数据同步

### 本地存储
- 浏览器 localStorage 保存一份
- 服务器文件系统保存一份
- 双重保障，更安全

### 离线模式
- 离线时数据存储在 localStorage
- 恢复网络后自动同步到服务器文件

## ⚠️ 注意事项

1. **不要手动编辑**: 直接修改JSON文件可能导致格式错误
2. **字符编码**: 文件使用UTF-8编码，支持中文
3. **文件权限**: 确保服务器有读写权限
4. **Git忽略**: `.gitignore` 已配置忽略 `*.json`，防止敏感数据提交
