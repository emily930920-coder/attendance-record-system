2. 重新运行 `npm start`
3. 数据文件仍然存在
4. 访问系统，所有记录都在

### 为什么不会丢失？

```javascript
// 服务器代码使用文件系统存储
const fs = require('fs').promises;
const filePath = path.join(__dirname, 'data', `${user}.json`);

// 每次操作都写入文件
await fs.writeFile(filePath, JSON.stringify(data, null, 2));
```

数据直接写入磁盘文件，**不在内存中**，所以：
- ✅ 重启服务 → 数据保留
- ✅ 关机重启 → 数据保留
- ✅ 断电恢复 → 数据保留（只要文件系统完好）

## ⚠️ Render免费版 - 会丢失数据

### 原因
Render免费版**没有持久化存储**：
- 每次部署/重启都是全新的容器
- 数据存储在临时文件系统
- 容器销毁时数据丢失

### 何时会丢失？
1. **重新部署**：推送新代码触发部署
2. **服务休眠后重启**：15分钟无访问后休眠
3. **手动重启**：在Dashboard手动重启服务
4. **系统维护**：Render平台维护

### 解决方案

#### 方案1: 升级到付费版（推荐）
```
Render付费版: $7/月
✅ 持久化存储
✅ 不会休眠
✅ 数据永久保存
```

#### 方案2: 使用外部数据库（免费）
连接免费数据库服务：
- MongoDB Atlas（免费500MB）
- Supabase（免费500MB PostgreSQL）
- PlanetScale（免费5GB MySQL）

#### 方案3: 定期备份（免费但需手动）
- 每天导出数据
- 保存到云盘/GitHub
- 需要时手动恢复

#### 方案4: 仅用于演示（接受数据丢失）
如果只是演示用途，可以接受数据会丢失

## 🐳 Docker部署 - 数据不会丢失（如果正确配置）

### 正确配置（数据持久化）

**docker-compose.yml**（项目已包含）：
```yaml
services:
  attendance-system:
    volumes:
      - ./server/data:/app/server/data  # 数据卷映射
```

这样配置后：
- ✅ 重启容器 → 数据保留
- ✅ 删除容器 → 数据保留
- ✅ 重建容器 → 数据保留

### 错误配置（数据会丢失）

如果没有配置数据卷映射：
```bash
# 错误示例 - 没有 -v 参数
docker run -p 3000:3000 attendance-system
```

这样会导致：
- ❌ 删除容器 → 数据丢失
- ❌ 重建容器 → 数据丢失
