## 🚀 部署步骤

### 步骤1: 选择备份服务

#### 选项A: Webhook.site（最简单，推荐）⭐

1. **访问**: https://webhook.site
2. **获取URL**: 页面自动生成唯一的Webhook URL
3. **复制URL**: 形如 `https://webhook.site/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`

**优势**:
- ✅ 完全免费
- ✅ 无需注册
- ✅ 实时查看数据
- ✅ 保留30天历史

**使用**:
```bash
BACKUP_WEBHOOK_URL=https://webhook.site/你的唯一ID
```

#### 选项B: Discord Webhook（免费，推荐）⭐

1. **创建Discord服务器**（如果没有）
2. **创建频道**用于接收备份
3. **获取Webhook**:
   - 频道设置 → 整合 → Webhooks
   - 创建Webhook
   - 复制Webhook URL

**优势**:
- ✅ 完全免费
- ✅ 移动端可查看
- ✅ 数据永久保存
- ✅ 可设置通知

**使用**:
```bash
DISCORD_WEBHOOK_URL=https://discord.com/api/webhooks/你的webhook
```

#### 选项C: Pipedream（高级，免费）

1. **注册**: https://pipedream.com
2. **创建Workflow**
3. **添加HTTP Trigger**获取Webhook URL
4. **配置存储**:
   - 保存到Google Sheets
   - 发送到邮箱

**优势**:
- ✅ 免费额度充足
- ✅ 可自定义处理逻辑
- ✅ 集成各种服务

#### 选项D: 邮件备份

使用邮件服务的Webhook API：
- **SendGrid**: https://sendgrid.com

- **Resend**: https://resend.com

## 📊 备份时间表

### 默认配置: 每天凌晨2点（UTC）

**对应北京时间**: 上午10点

**Cron表达式**:
```
0 2 * * *  # 每天凌晨2点（UTC）
```

### 自定义备份频率

修改 `render.yaml` 中的 `schedule`:

```yaml
# 每12小时一次
schedule: "0 */12 * * *"

# 每6小时一次
schedule: "0 */6 * * *"

# 每天凌晨和中午
schedule: "0 2,14 * * *"

# 每周一凌晨2点
schedule: "0 2 * * 1"
```

## 🛠️ 高级配置

### 1. 备份到Google Sheets

使用Pipedream配置：

```javascript
// Pipedream Workflow
export default defineComponent({
  async run({ steps, $ }) {
    const data = steps.trigger.event.body;
    
    // 保存到Google Sheets
    await $.send.http({
      method: 'POST',
      url: 'https://sheets.googleapis.com/v4/spreadsheets/...',
      data: data
    });
  }
});
```

### 2. 备份到GitHub Gist

```javascript
// 在backup.js中添加
async function backupToGist(data) {
    const response = await fetch('https://api.github.com/gists', {
        method: 'POST',
        headers: {
            'Authorization': `token ${process.env.GITHUB_TOKEN}`,
            'Content-Type': 'application/json'
        },
        body: JSON.stringify({
            description: `工时记录备份 - ${new Date().toISOString()}`,
            public: false,
            files: {
                'backup.json': {
                    content: JSON.stringify(data, null, 2)
                }
            }
        })
    });
    return response.ok;
}
```

## ❓ 常见问题

### Q1: 备份失败怎么办？

**A**: 检查以下几点：
1. 环境变量是否正确配置
2. Webhook URL是否有效
3. 查看Render Logs中的错误信息
4. 即使失败，数据也会输出到日志

### Q2: 如何验证备份是否成功？

**A**: 
- Webhook.site: 实时查看请求
- Discord: 检查频道是否有新消息
- Render Logs: 搜索"备份成功"

### Q3: 备份的数据安全吗？

**A**: 
- Webhook.site: 30天后自动删除
- Discord: 建议设置私有频道
- 可以加密数据再备份

### Q4: 可以改变备份时间吗？

**A**: 可以，修改 `render.yaml` 中的 `schedule`

### Q5: 本地部署也可以用这个备份吗？

**A**: 可以！运行 `npm run backup` 即可
