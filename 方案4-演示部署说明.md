   ```

   Environment: Node
   Branch: main
   Build Command: npm install
   Start Command: npm start
   Instance Type: Free
   ```

5. **部署**
   - 点击 "Create Web Service"
   - 等待3-5分钟部署完成

## 💡 最佳实践

### 1. 使用UptimeRobot保持活跃

**目的**: 防止15分钟休眠

**步骤**:
1. 注册 https://uptimerobot.com（免费）
2. 添加监控:
   - 类型: HTTP(s)
   - URL: https://你的服务名.onrender.com/api/health
   - 间隔: 5分钟
3. UptimeRobot会定期访问，保持服务活跃

## ❓ 常见问题

### Q1: 演示时服务很慢怎么办？

**A**: 免费版服务休眠后首次访问需要30-60秒启动。

**解决**:
- 演示前5分钟访问一次
- 使用UptimeRobot保持活跃
- 准备演示视频备用

### Q2: 数据突然没了怎么办？

**A**: 这是正常的，服务重启会清空数据。

**解决**:
- 提前说明这是演示环境
- 重新添加示例数据
- 或展示本地部署版本

### Q3: 可以导入我的真实数据吗？

**A**: 可以，但不推荐。

**原因**:
- 数据会丢失
- 如有敏感信息不安全
- 建议使用示例数据

### Q4: 如何升级到持久化存储？

**A**: 随时可以升级：

1. **Render付费版**: $7/月
   - Dashboard → Settings → Upgrade
   - 添加Persistent Disk

2. **MongoDB Atlas**: 免费
   - 按照方案2实施
   - 需要改造代码
